Everything You Need for Application Delivery

Paprika consolidates the tools your team needs into a single operator — no more stitching together Argo projects.

Unified Resource Model

Replace ArgoCD, Rollouts, and Workflows with a single Application CRD. One operator, one resource model, one consistent API.

Progressive Delivery

Canary and blue-green deployments with configurable step weights. Let traffic shift gradually and verify health at every step.

Pluggable Traffic Routing

Built-in support for Istio VirtualServices and Gateway API HTTPRoutes. Per-stage traffic router configuration for fine-grained control.

Multi-Cluster by Default

Stage-level cluster references let you deploy to different clusters per environment. No pre-configured cluster secrets required.

Health Verification

CEL-based health checks, built-in resource health rules, and approval gates ensure deployments are verified at every stage.

Built-in Dashboard

Real-time visibility into pipelines, releases, stages, and applications. Connect-RPC API for programmatic access.

How It Works

From source code to production — in three steps.

Step 1

Define Your Application

Create an Application resource that references your Helm chart, Git repo, or S3 bucket. Specify build steps, stages, and deployment strategy in a single YAML manifest.

apiVersion: pipelines.papriko.io/v1alpha1
kind: Application
metadata:
  name: my-app
spec:
  source:
    git:
      url: https://github.com/example/app
      revision: main
  strategy: Canary
Step 2

Promote Through Stages

Paprika renders manifests using Helm SDK, runs pipeline steps as Kubernetes Jobs, and promotes releases through stages — each with its own cluster, canary config, and traffic routing.

  stages:
    - name: staging
      canary:
        steps:
          - weight: 10
          - weight: 50
          - weight: 100
    - name: production
      gates:
        - name: approve
          type: ManualApproval
Step 3

Verify and Release

Health checks evaluate every resource at each stage. Traffic routing shifts weights gradually. Approval gates pause for manual verification. Paprika handles the rest.

  stages:
    - name: production
      trafficRouter:
        istio:
          virtualService: my-app
          host: app.example.com
