Getting Started

Install Paprika on a Kubernetes cluster and deploy your first application in under 5 minutes.

Prerequisites

  • Kubernetes cluster v1.29+
  • cert-manager installed (webhook TLS certificates)
  • kubectl configured for your cluster
  • Docker (for building the operator image)

Installation Options

Option 1: Deploy from Source

git clone https://github.com/benebsworth/paprika.git cd paprika # Install CRDs make install # Build and push to your registry export IMG=ghcr.io/YOUR_USER/paprika:latest make docker-build docker-push IMG=$IMG # Deploy the operator make deploy IMG=$IMG # Verify kubectl get pods -n paprika-system

Option 2: Single YAML Bundle

make build-installer IMG=ghcr.io/YOUR_USER/paprika:latest kubectl apply -f dist/install.yaml

Option 3: Helm Chart

make helm-generate helm install paprika charts/chart --namespace paprika-system --create-namespace

Deploy a Sample Application

Create an Application resource:

kubectl apply -f - <<EOF apiVersion: pipelines.paprika.io/v1alpha1 kind: Application metadata: name: my-app namespace: paprika-system spec: source: type: helm chart: repo: https://charts.bitnami.com/bitnami name: nginx version: 18.2.2 stages: - name: dev ring: 1 strategy: Rolling syncPolicy: Auto EOF

What Happens

When you create the Application, Paprika's controllers automatically:

  1. Template Created — A Template resource is created from the source field
  2. Stage Created — A Stage resource is created for each stage entry
  3. Release Created — A Release promotes the rendered manifests through the stage lifecycle
  4. Manifests Rendered — The Helm chart is rendered using the Helm SDK
  5. Manifests Applied — Rendered resources are applied to the cluster
  6. Health Verified — The operator monitors resource health via CEL expressions

Check Status

kubectl get application my-app -n paprika-system -o yaml # Watch the phase transition: Pending → Promoting → Healthy kubectl get application my-app -n paprika-system -w

Access the Dashboard

The dashboard is served by the operator on port 3000:

kubectl port-forward -n paprika-system deployment/paprika-controller-manager 3000:3000 # Open http://localhost:3000

Run Locally (Development)

ENABLE_WEBHOOKS=false make run

This runs the operator on your host machine using your current kubeconfig context. Webhooks are disabled for local development.

Run Tests

# Unit tests with envtest make test # Lint make lint # E2E tests (creates an isolated Kind cluster) make test-e2e

Clean Up

# Delete your application kubectl delete application my-app # Uninstall the operator make undeploy # Remove CRDs make uninstall
