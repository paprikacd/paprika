We're excited to announce the first release of Paprika — **v0.1.0**. This initial release focuses on the core operator functionality with a dashboard UI for visibility into your deployments.

## What's Included

### Six Custom Resource Definitions

- **Application** — the top-level resource that composes sources, pipelines, stages, and releases
- **Template** — source configuration supporting Helm charts, Git repositories, and S3 buckets
- **Pipeline** — sequential build/test/deploy steps executed as Kubernetes Jobs
- **Stage** — environment definitions with cluster refs, canary configuration, and traffic routing
- **Release** — promotion lifecycle through stages with health verification and approval gates
- **Artifact** — build outputs passed between pipeline steps

### Traffic Routing

This release includes two traffic router providers:

| Provider      | Resource        | Status     |
| ------------- | --------------- | ---------- |
| Istio         | VirtualService  | Stable     |
| Gateway API   | HTTPRoute       | Stable     |

Both support weighted traffic splitting for canary deployments, with the traffic weight set per-stage in the `StageSpec`.

### Health Evaluation

Paprika's health engine evaluates resource health using:

- **CEL expressions** — custom health checks written in Kubernetes' Common Expression Language
- **Built-in health rules** for Deployments (replica availability), Services, ConfigMaps, and Secrets
- **Custom resource health** — extensible via the `ResourceHealthChecker` interface

### Dashboard UI

The built-in dashboard provides a real-time view of your pipelines, releases, stages, and applications:

- Live polling via Connect-RPC
- Connection status indicator
- Per-section error handling with retry
- Skeleton loading states
- Dark mode by default

### Installation Options

Paprika can be installed three ways:

1. **Helm chart** — recommended for production deployments
2. **Kustomize bundle** — `kubectl apply -f dist/install.yaml`
3. **Source build** — `make build && make deploy`
